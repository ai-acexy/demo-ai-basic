### LLM对话模型是如何演变到AI应用（Agent智能体）

LLM：就像一个全身瘫痪的智者，拥有全knowledge，能基于已有知识反馈，能基于给定的资料推理，但是只停留在问答阶段。

Agent：就像给LLM装上手脚。为了解决一个问题，让他能够自己执行，操作，控制所有可用的资源。

---

#### Chat聊天方式解决问题 user <-> llm

通过一问一答的方式，获取答案。

局限性:

- 需要多次描述问题 -> 引入Prompt Context Memory
- 模型没有最新的数据 -> 引入Search

#### Agent智能体操控解决问题 user <-> agent <-> llm

通过提出问题，Agent分析，自主计划拆解任务，操作相关资源，汇总相关资料，获取答案。

优势：

- 解决用户问题时，模拟人类解决问题的流程，可以自主拆解任务，分步实现，思考验证观察。
- 可以与实体功能进行交互，可以产生实际可感知的非文本结果。
