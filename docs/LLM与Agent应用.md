### 从 LLM 对话模型到 AI 应用（Agent 智能体）

**LLM**：像一个拥有渊博知识但全身瘫痪的智者——能基于已有知识作答，能对给定材料进行推理，但止步于文本问答。

**Agent**：就像给这位智者装上了手脚，让他能够自主规划、执行操作、调度一切可用资源，以解决实际问题。

---

#### Chat 模式：对话解决问题

```
用户 <--> LLM
```

通过一问一答直接获取答案。

**局限性：**

- 需反复描述问题和背景 → 引入 Prompt / Context / Memory 机制
- 模型知识存在截止日期，缺乏最新数据 → 引入 Search

#### Agent 模式：智能体操控解决问题

```
用户 <--> Agent <--> LLM
```

用户提出问题后，Agent 分析需求、自主拆解任务、调度相关资源、汇总信息，最终给出答案。

**优势：**

- 模拟人类解决问题的方式：拆解任务 → 分步执行 → 思考验证 → 观察修正
- 可与外部系统交互（API、数据库、文件系统等），产生非文本的实际操作结果
