### 常见基本名词释义

- **LLM（大语言模型）**

  基于 Transformer 架构的大规模预训练语言模型，例如 GPT、Claude、Llama 等。本质上是一个"概率预测引擎"，根据上文预测下一个 token。模型本身无状态、无外部工具调用能力、无持久记忆。

- **Prompt（提示词）**

  发送给模型的输入文本，用于约束和指导模型的输出行为。常见作用：设定角色、约束输出格式、引导推理方式、明确任务目标。由于 LLM 本身无状态，持续对话是通过每次将历史 prompt 重新发送实现的。

- **Context（上下文）**

  模型单次推理时可见的全部输入内容，包括 prompt、历史对话、外部注入数据等。上下文窗口大小决定了模型能"看到"的信息量上限，直接影响回答的知识范围和推理深度。

- **Memory（记忆）**

  为实现跨轮次对话的连续性，将关键上下文信息持久化保存，并在后续请求中按需注入 context 的机制。通常分为短期记忆（当前会话窗口内）和长期记忆（外置存储、向量库等）。

- **Agent（智能体）**

  能自主决策、规划、调用工具以完成复杂任务的 AI 系统/组件。典型工作流程：
  - 理解需求
  - 拆解目标
  - 选择工具并执行
  - 验证执行结果
  - 迭代修正
  - 输出最终结果

  根据设计目的，Agent 不必须完整实现上述所有环节——最小闭环仅需输入 → 输出。

- **Search（搜索）**

  接入外部数据源（搜索引擎、数据库等）获取实时或私有数据，解决模型知识截止日期滞后、缺少特定领域数据的问题。

- **RAG（检索增强生成）**

  实现 Search 的主流方案之一。预先将私有知识库向量化并建立索引，查询时检索相关文档片段，注入 prompt 交由 LLM 处理生成回答。

- **MCP（模型上下文协议）**

  由 Anthropic 主导设计的开放标准协议，用于规范 LLM 应用与外部工具/数据源之间的交互方式。开发者按 MCP 规范实现工具，即可被各类 AI 平台统一集成调用。

- **Skills（技能）**

  提示词管理方案：将特定领域的功能、知识和约束封装为可复用的描述文件。构建通用 AI 助手时，可定义不同领域的标准化提示词模板，实现按需激活。

- **Tool（工具）**

  LLM 本身只能输出文本。Tool 让模型可以调用外部函数（API、数据库、文件系统等）执行实际操作。模型在推理时输出"函数调用请求"，由宿主程序执行后将结果返回模型继续处理。
