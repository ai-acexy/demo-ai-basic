### 常见基本名词释义

- LLM

  大语言模型，基于 Transformer 架构的大规模预训练模型，例如 GPT 系列、Claude、Llama 等。一个“概率分布预测引擎”，根据上下文预测下一个
  token。它本身没有外部能力，也没有长期记忆。
- Prompt

  提示词，由于大模型的无状态性，为了实现持续对话或者指导模型的具体运行时的指令的文本约束。常见作用，指导模型角色，约束输出格式，引导推导方式，给定任务目标。

- Context

  模型一次推理时可见的全部输入内容（含prompt，历史对话，外部注入数据），可以决定模型回答的知识范围，推理深度，记忆可见性。
- Memory

  为持续性的会话，将重要的上下文信息保存起来，用于持续发送给大模型，形成连续对话。分为长期+短期。用于维持会话连续性，跨任务等。
- Agent

  智能体（代理）能自主决策，规划，调用工具，完成复杂任务的基于AI的一个``系统``/``功能组件``。
    - 可以理解需求
    - 指定拆解目标
    - 选择工具执行
    - 验证执行步骤
    - 反复推导步骤
    - 输出最终结果

  基于设计它的目的，智能体并不需要完整实现上述功能，仅需实现部分功能即可，最低架构为 输入->输出。
- Search

  接入外部数据源，搜索实时数据等。解决模型知识库过期或需要使用额外数据来让LLM处理。
- RAG

  解决Search的方案之一，检索搜索增强，构建私有的数据物料，相关内容可以通过RAG搜索注入prompt发送给LLM处理。
- MCP

  由Anthropic主导设计的一个标准，用于定义构建大模型AI应用时，开发者在运行/发布工具时，具有标准的规范。大家可以构建不同功能的MCP，然后方便，统一的集成使用。
- Skills
  
  提示词管理方案，将一部分通用的功能，结合组装，通过说明文件来描述作用。构建通用型AI助手时，可以定义不同领域，不同要起的通用默认的标准提示词。
